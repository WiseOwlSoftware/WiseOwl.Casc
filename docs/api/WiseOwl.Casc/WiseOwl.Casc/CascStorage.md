# CascStorage class

An opened, local Blizzard CASC storage. This is the game-agnostic transport entry point: it resolves content by [`EncodingKey`](./EncodingKey.md) or [`ContentKey`](./ContentKey.md), reads the archive envelope, and BLTE-decodes it. Game-specific name/SNO resolution lives in the game modules (e.g. `WiseOwl.Casc.Diablo4`).

## Remarks

Reads only a local installation (no CDN). The encoding table is large, so it is parsed lazily on first content-key use; the local index is parsed up front. Instances are thread-safe for reads.
