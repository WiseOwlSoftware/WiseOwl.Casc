# Md5Key structure

A 128-bit CASC key (the MD5-derived hash CASC uses to address content). Stored as two UInt64 halves so the type is a small, allocation-free value with O(1) equality on every target framework (including `netstandard2.0`, which lacks inline arrays).

## Remarks

This is the shared representation behind the strongly-typed [`ContentKey`](./ContentKey.md) and [`EncodingKey`](./EncodingKey.md). The two public wrappers exist so the compiler stops you handing an encoding key to an API that wants a content key — a real source of bugs in older CASC libraries that used raw `byte[]` everywhere.
