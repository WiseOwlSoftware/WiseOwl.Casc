## Remarks

Unlabeled columns are exposed raw, not renamed. The 212-byte row has ~10 non-zero columns, but only `+4` ([`HpScalar`](./LevelScalingRow/HpScalar.md)) is oracle-anchored (it drives base Max Life, §8.2). The Maxroll dump names the others `monsterDr` / `powerBase` / `powerDelta` / `powerItem` / `xpScalar`, but — unlike `DifficultyTiers`'s XP column (§8.3), which has an independent anchor — none of these can be verified from the blob (no anchor, no in-game oracle), so this library does not assert those names. Their per-level behavior is characterised in §8.2; they are exposed by byte offset on [`Columns`](./LevelScalingRow/Columns.md) so consumers have every column without a label that can't be stood behind.
