# AssetKind enumeration

FR-C20 — the family an [`AssetRef`](./AssetRef.md) belongs to. Each kind is backed by one provider that knows how to enumerate its [`AssetRef`](./AssetRef.md)s from `CoreTOC` and decode one into its strongly-typed definition. Kinds flagged as render recipes (the draw/composition surface) are also reachable via [`RenderRecipesOnly`](./AssetQuery/RenderRecipesOnly.md).  Adding a new family is one new provider + one value here — no change to the [`Catalog`](./Catalog.md) facade. Sub-classification within a kind (e.g. a weapon vs armor Item) rides on [`Tags`](./AssetRef/Tags.md), not on new kinds.
