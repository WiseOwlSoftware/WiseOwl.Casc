# PowerScriptFormula structure

FR-C13 Phase 1 — one slot of a Power's Script Formula table. Each slot is a positional, named entry the engine resolves through its runtime formula evaluator; the library surfaces the entry's decoded text form + literal IEEE-754 value (when the text is a numeric literal) so the consumer can evaluate the SF_N references in [`Description`](./PowerDefinition/Description.md)'s format string.

```csharp
public struct PowerScriptFormula : IEquatable<PowerScriptFormula>
```

| parameter | description |
| --- | --- |
| Index | Positional index — matches the engine's `[SF_n...]` placeholders in the localized [`Description`](./PowerDefinition/Description.md) (e.g. `[SF_0*100|%x|]` references [`Index`](./PowerScriptFormula/Index.md) = 0). |
| Text | The decoded text form of the slot's `DT_STRING_FORMULA.Value`. For trivial-numeric slots this is the literal text (`"0.02"`, `"60"`, `".15"`, …); for expression slots this is an arithmetic expression over other SF_N references (`"SF_1 / 3"` on Demonic Spicules' SF_2). |
| LiteralValue | The engine's IEEE-754 single-precision scalar for the slot — populated for trivial-numeric slots from the compiled-form float; NaN for expression slots (Phase 2 will resolve those library-side). |
