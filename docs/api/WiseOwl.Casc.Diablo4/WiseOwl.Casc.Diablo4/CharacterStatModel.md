# CharacterStatModel class

Universal (class-independent) constants for the Diablo IV Character-Sheet stat model (FR-C29). The per-point core→stat conversion coefficients and the inherent base stats were not located in any searched SNO source (a thorough data-mine across every candidate GameBalance/class source found no coefficient home — see `docs/casc-diablo4-format.md §12`): they are either engine-side or in a global config not yet identified. They are universal — identical for every class, so a newly-added class reuses them and supplies only its own data-driven core→stat map (which is why they need not be per-class data). They are baked here as owner-oracle-validated constants (the engine-constants pattern; precedent CL-68 / CL-83), pinned against live core-stat tooltips for four classes spanning all four primary-attribute archetypes (Warlock, Rogue, Necromancer, Barbarian), including a high-Paragon capture that fixed the small-magnitude rates to three significant figures.

## Remarks

The per-class core→stat mapping (which core feeds Skill Damage / Critical Strike Chance / Resource Generation) is data-driven and is decoded structurally per class by [`PlayerClassDefinition`](./PlayerClassDefinition.md) — no class table is hard-coded. Re-verify trigger (Appendix D): a new game build whose core-stat tooltips disagree with these rates.
