# ParagonBoardGrid constructor

FR-C17 — the engine's paragon-board grid-layout metric, in the authored design-canvas reference units. The consumer maps a board cell's grid coordinate to a canvas position with `canvasPos = (gridX·Pitch, gridY·Pitch)` (relative to the grid origin) and scales the whole canvas `CanvasWidth×CanvasHeight → render resolution`. This replaces the consumer's empirical pitch (the FR-C7-measured ~67.7px / arbitrary CellPx) with the engine's authored metric.  All values are read from the live UI scene (game data), not hard-coded: [`CanvasWidth`](./CanvasWidth.md)/[`CanvasHeight`](./CanvasHeight.md) from the root `ParagonBoard_main` widget, [`CellExtent`](./CellExtent.md) from the `Template_Node_Common` node-cell widget. The board's logical grid (dimensions + which cell holds which node) is the per-board [`ParagonBoardDefinition`](../ParagonBoardDefinition.md) (`Width` + `Cells`); this record is the global canvas/cell metric shared across boards. Pitch = CellExtent (cells are laid out adjacent, no extra inter-cell gap beyond the per-cell art inset). Validated against the owner's in-game measurement: the empirical ~67.7px pitch = `CellExtent (100) × render-scale (≈0.677 at the consumer's board width)`, so the authored 100-unit cell reproduces the observed spacing exactly. If a future build introduces an explicit inter-cell gap it would surface as `Pitch ≠ CellExtent` (engine render-code RE — the `UIParagonBoardStyle` class is a style wrapper with no grid-layout fields, so any gap lives in the engine's grid-layout code, not the scene data).
