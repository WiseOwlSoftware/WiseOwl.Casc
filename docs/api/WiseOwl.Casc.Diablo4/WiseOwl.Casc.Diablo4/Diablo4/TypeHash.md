# Diablo4.TypeHash method

The Diablo IV type hash: the same DJB2 core as [`GbidHash`](./GbidHash.md) (`h = 0; h = h*33 + c`, unsigned 32-bit, stopping at the first NUL) but case-sensitive (no lower-casing). D4 identifies serialized type / class / struct names with this — e.g. `DT_INT`, `DT_BINDABLEPROPERTY`, and the UI widget class ids in the `0xE4825AB8` UI-scene format.

```csharp
public static uint TypeHash(string name)
```

| parameter | description |
| --- | --- |
| name | The type / struct name (ASCII, case-sensitive). |

## Remarks

Same algorithm family as [`GbidHash`](./GbidHash.md) (which lower-cases first) and [`FieldHash`](./FieldHash.md) (which masks to 28 bits); all three are seed-0 DJB2, not the textbook seed 5381. See `docs/casc-diablo4-format.md §10.2`. Verified: `TypeHash("DT_INT") == 0xA4C42E02`, `TypeHash("DT_BINDABLEPROPERTY") == 0x1332C78D`.
