# ItemType.EItemType property

The engine `eItemType`ordinal — the value that appears in an affix's [`AllowedItemTypes`](../AffixDefinition/AllowedItemTypes.md) pool (#51, CL-108). Decoded as the first entry of the record's item-type array (the `DT_VARIABLEARRAY` at payload `+0x28`). This is the missing link that names an affix pool: e.g. `Helm`=16, `ChestArmor`=17, `Gloves`=28, `Boots`=29, `Legs`=30, `Amulet`=26, `Ring`=19, `Axe`=1, `Bow`=10, `Charm`=71 (build 3.1.1.72836). 1H/2H and class variants share one ordinal (`Axe` and `Axe2H` are both `1`) — the ordinal is the base type, so a name lookup is one-ordinal-to-many-names. −1 when the record carries no item-type array (non-gear types) or it is malformed. A small number of ordinals that appear in affix pools (e.g. `9`, `23`) have no g98 [`ItemType`](../ItemType.md) record and so cannot be named from the data — they are engine-aggregate/legacy values; use [`GetItemTypeName`](../Diablo4Storage/GetItemTypeName.md) which returns `null` for those.
