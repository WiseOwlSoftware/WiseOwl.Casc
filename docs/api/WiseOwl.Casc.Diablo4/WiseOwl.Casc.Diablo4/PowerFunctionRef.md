# PowerFunctionRef structure

FR-C13 Phase 2 — one engine-function reference that appears in a Power's `DT_STRING_FORMULA` text expressions or in its localized [`Description`](./PowerDefinition/Description.md) format string. The library surfaces the reference structurally (name + resolved arg values); the consumer registers a resolver delegate keyed by [`Name`](./PowerFunctionRef/Name.md) to substitute the engine value at runtime (engine-runtime player-state is outside CASC's domain).

```csharp
public struct PowerFunctionRef : IEquatable<PowerFunctionRef>
```

| parameter | description |
| --- | --- |
| Name | The engine-function identifier as it appears in the expression text (e.g. `"PlayerHealthMax"`). Case-sensitive match with whatever identifier convention the engine uses. |
| Args | The resolved argument values (each evaluated as a Double) the engine passes to the function call. Empty for zero-argument calls (the common case for engine state accessors). |
