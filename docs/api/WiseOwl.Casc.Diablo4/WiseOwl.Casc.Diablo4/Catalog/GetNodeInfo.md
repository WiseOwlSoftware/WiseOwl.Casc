## Remarks

Caching. Results are cached by SNO for the storage's lifetime — the optimizer hot path re-queries the same boards repeatedly, and each board carries ~17–21 distinct node defs across ~441 cells. Decode once, reuse. The cache also stores negative results (missing/undecodable SNOs) so a malformed repeat-query is just as cheap as a hit.
