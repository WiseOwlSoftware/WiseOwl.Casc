## Remarks

Performance contract. The board itself is cached (one decoded [`ParagonBoardDefinition`](../ParagonBoardDefinition.md) per SNO), the per- cell [`ParagonNodeInfo`](../ParagonNodeInfo.md)s are cached per node SNO (~17–21 distinct per board, ~441 cells), and the projected `(cell, info)` list is cached per board SNO so repeat queries return the same instance with O(1) cost. Missing / undecodable board SNOs memoize as an empty list (the optimizer's search-tree pruning often probes malformed ids; the cache makes re-probes free).

Ordering. Row-major (`row 0 = top`, `col 0 = left`), skipping empty cells.
