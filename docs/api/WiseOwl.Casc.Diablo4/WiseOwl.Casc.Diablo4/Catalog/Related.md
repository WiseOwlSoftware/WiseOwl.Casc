# Catalog.Related method

FR-C20 P5 — the asset's authored relationship edges, so board → node → power and glyph → affix / class traversal is first-class instead of hand-assembled. Each [`Target`](../AssetLink/Target.md) is a full [`AssetRef`](../AssetRef.md) — call `Related` on it to chain. Authored FK edges only:

* ParagonBoard → `node` (distinct non-empty cells).
* ParagonNode → `power` (its `SnoPassivePower`, when set — the legendary-node passive).
* ParagonGlyph → `affix` (granted) + `class` (usable-by).

A socket node ↔ glyph is a runtime slotting, not an authored FK, so it is deliberately not a link; find candidate glyphs for a class via [`FindByFacet`](./FindByFacet.md).
