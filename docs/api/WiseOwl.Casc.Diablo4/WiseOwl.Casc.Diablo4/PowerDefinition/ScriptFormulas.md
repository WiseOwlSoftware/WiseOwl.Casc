# PowerDefinition.ScriptFormulas property

FR-C13 Phase 1 — the positional Script Formula slot table decoded from the Power record's tail data. Each entry is one `DT_STRING_FORMULA`'s compiled form: a text form (numeric like `"0.02"` or an arithmetic expression like `"SF_1 / 3"`) and the IEEE-754 literal value (the engine's pre-computed scalar when the text is numeric; NaN when the text is an expression). Indexed positionally: [`Index`](../PowerScriptFormula/Index.md) matches the `[SF_n...]` placeholders in [`Description`](./Description.md) (the engine's format-string SF_N indices).  Empty when the power has no script formulas (most active skills and many structural / informational powers fall here). Non-empty for the 72 legendary node powers + others using the `DT_STRING_FORMULA` mechanism. The trailing footer records (`"10"` / 10.0 followed by `"0"` / 0.0 on every anchored legendary) are stripped from this list — they are a fixed value-descriptor footer, not SF_N values and not the rank cap (FR-C38 corrected the CL-100 misread; the per-aspect max rank is [`MaxRank`](../AffixDefinition/MaxRank.md), the int32 at affix payload `+0x94`).
