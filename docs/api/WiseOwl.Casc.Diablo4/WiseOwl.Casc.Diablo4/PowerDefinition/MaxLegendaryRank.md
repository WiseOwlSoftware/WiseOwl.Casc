# PowerDefinition.MaxLegendaryRank field

The universal maximum legendary/aspect rank (LIB-3 R7, CL-100) — the engine-global cap that `CurrentLegendaryRank()` reaches in the rank-scaled value formulas of legendary/unique item aspects (see [`InlineFormula`](../AffixEffect/InlineFormula.md)). 10 on the live build (`3.1.1.72836`, Season 14) and verified universal — every one of the 699 `legendary_*` Power records carries an identical `("10", 10.0)` max-rank sentinel in its script-formula tail, with zero exceptions (owner-confirmed as the max-tier/rank cap, FR-C13 R2).  It is a constant rather than a per-aspect field because the max rank is not stored on the affix record — an aspect modifier carries only its inline value formula, and the rank cap is an engine global. The live-decoded backing is [`MaxRank`](./MaxRank.md) (the value read from the sentinel on any legendary Power); this constant equals that decode and exists so consumers computing an aspect's rank span (`[formula(1) … formula(MaxLegendaryRank)]`) need not read a Power. Per the engine-constants pattern (`casc-diablo4-format.md` Appendix D) it carries a seasonal re-verify trigger: a build bump re-runs `SnoScan ranksentinel` and updates this if the cap ever changes.
