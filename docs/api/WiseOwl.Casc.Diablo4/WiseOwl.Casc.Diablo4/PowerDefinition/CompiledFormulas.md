# PowerDefinition.CompiledFormulas property

FR-C13 Phase 3 — the engine's compiled-form `{SF_N → value}` map, derived from the binary `DT_STRING_FORMULA` records' IEEE-754 floats (for literal slots) and from binary AST opcode operands (for expression slots). The values are the engine-truth: the IEEE-754 singles the engine literally executes, promoted to Double.  Distinct from [`ResolvedFormulas`](./ResolvedFormulas.md) only when the engine has compiled a slot's text form (e.g. `"0.02"` → `0.02f`) inconsistently — in that case the binary IEEE-754 single is canonical and this map reflects it, while [`ResolvedFormulas`](./ResolvedFormulas.md) reflects the text-parsed value. In normal operation the two maps agree to float precision for all 9 Warlock legendary anchors (the FR-C13 R5 regression gate).  For expression slots (Demonic Spicules's `SF_2 = "SF_1 / 3"` is the canonical anchor): the operator is parsed from the slot's text, but numeric operands are substituted from the binary AST opcode region's embedded IEEE-754 singles in encounter order. So Demonic Spicules's `SF_2` evaluates as `SF_1 / binary_literal[0]` = `60 / 3.0f` = `20`; the `3.0f` comes from the compiled record's binary bytes, not from re-parsing the text `"3"`.  AST opcode encoding for non-literal sub-trees (e.g. function calls inside expression records) is not yet fully decoded (devlog 0035 §3). For powers whose compiled AST encoding is outside the current decoder's scope, the slot is left at NaN — the consumer compares non-NaN entries for cross-validation.
