# ParagonNodeStat constructor

One stat grant on a paragon node — the display-ready projection of a [`NodeAttribute`](../NodeAttribute.md). The library does the formula evaluation and unit inference for the node-info surface (FR-C21 carve-out from the Appendix C boundary).

```csharp
public ParagonNodeStat(int AttributeId, string StatName, int Variant, string? VariantName, 
    double? FlatValue, StatUnit Unit, AssetRef? Formula, string? InlineFormula)
```

| parameter | description |
| --- | --- |
| AttributeId | The raw [`AttributeId`](../NodeAttribute/AttributeId.md) — the engine's `eAttribute`. Note: this is a power-budget category, not the canonical stat key (three distinct [`Sno`](../ParagonNodeInfo/Sno.md)s can share id `481`; see the Optimizer correction relayed in `docs/devlog/0063-*.md`). Surfaced raw so the consumer can inspect the budget category — for stat-identity aggregation, key on [`Sno`](../ParagonNodeInfo/Sno.md). |
| StatName | Human-readable stat name, derived clean-room from the `Generic_<Rarity>_<Token>` node-name convention (light CamelCase split + known-abbreviation expansion; `Generic_Magic_DamageToElite` → `"Damage to Elite"`). Non-Generic class-specific node names (e.g. `Warlock_Rare_006`) have no encoded stat token and fall back to `"Attribute <id>"`. |
| Variant | The raw [`NParam`](../NodeAttribute/NParam.md) — informational. Always `0` on every paragon stat node sampled so far (the per-element resistance/damage variants the engine distinguishes via a single [`NParam`](../NodeAttribute/NParam.md) in other domains are split into distinct nodes here, so each carries its own [`Sno`](../ParagonNodeInfo/Sno.md) with the variant expressed in the node name). |
| VariantName | Reserved (always `null` today). Iterate via a follow-on FR if a future stat surfaces a non-zero *Variant* with a meaningful resolution. |
| FlatValue | The displayed magnitude — the formula text from [`AttributeFormulaTable`](../AttributeFormulaTable.md) evaluated via [`Evaluate`](../ParagonMagnitudeFormula/Evaluate.md) (or the inline constant on a Normal-rarity node). `null` when the formula references an unknown budget-multiplier intrinsic (a future build added one the calibration table hasn't picked up yet) — the consumer can still inspect *Formula*. |
| Unit | How to render *FlatValue* — see [`StatUnit`](../StatUnit.md). |
| Formula | Informational reference to the named [`AttributeFormulaTable`](../AttributeFormulaTable.md) entry the magnitude evaluation resolved through (AttributeFormulas with the formula name in [`Tags`](../AssetRef/Tags.md)); `null` when the node ships an inline formula ([`IsInline`](../NodeAttribute/IsInline.md)) — the inline text is on *InlineFormula*. |
| InlineFormula | The node's own inline formula text when [`IsInline`](../NodeAttribute/IsInline.md) is true; `null` when the node references a shared GameBalance formula via *Formula*. |
