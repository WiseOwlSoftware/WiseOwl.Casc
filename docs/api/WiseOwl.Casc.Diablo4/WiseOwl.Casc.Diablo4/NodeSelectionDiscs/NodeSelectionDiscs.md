# NodeSelectionDiscs constructor

FR-C16 R5/R9 — the per-selection-state disc layer pair of a rarity sub-template (`Template_Node_Magic`/`Rare`/`Legendary`). Structural claim: each rarity template is a parent widget whose anonymous child sub-records (each a `UIWindowStyle` + `0xFFFFFFFF` marker, name-less) bind one disc handle apiece; the first two handle-bearing children, in scene order, are [`Unselected`](./Unselected.md) then [`Selected`](./Selected.md). Role claim (separate): that child[0]=unselected and child[1]=selected (with the red perimeter ring baked into the selected disc) is the owner's FR-C12 rarity-disc oracle, which matches all six handles across the three rarities (`0x621CB6FF`/`0x72C29402` Magic, `0xB71BD068`/`0x03EDABAB` Rare, `0x232DF7F9`/`0xBD27FB7C` Legendary). The consumer draws [`Selected`](./Selected.md) for the currently-selected node and [`Unselected`](./Unselected.md) for every other node of that rarity; never both. Placement (FR-C18): the disc pair is co-sized — the engine authors the inset once (Magic/Rare carry it on the selected child: inset 7, i.e. the same 86² as the `Node_IconBase` grey base disc; Legendary carries inset −3 overscan on both). The pair's [`Rect`](../NodeDiscLayer/Rect.md)s are resolved here so both states share that authored rect: a colored rarity disc is the same on-screen size as the grey base disc it substitutes — not a full-cell blit. (The `Template_Node_<rarity>`parent widget binds no usable rect of its own; the all-zero parent rect FR-C18 reported is faithful — the disc geometry lives on these children.)
