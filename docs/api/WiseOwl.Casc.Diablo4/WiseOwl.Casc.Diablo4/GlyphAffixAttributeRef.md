# GlyphAffixAttributeRef structure

A single `(AttributeId, ParamPlus12)` entry from a glyph affix's [`AffectedAttributes`](./ParagonGlyphAffixDefinition/AffectedAttributes.md) list. The shape parallels the `ptAttributes` pair carried on a [`ParagonNodeDefinition`](./ParagonNodeDefinition.md) ‚Äî an `eAttribute` int and a `uParamPlus12` GBID ‚Äî but the glyph-affix encoding is the trimmed 8-byte pair only (no inline formula, no parallel attribute-GBID, no `NParam`), so it is surfaced as a distinct record struct rather than reusing [`NodeAttribute`](./NodeAttribute.md)'s 88-byte node-attribute shape.

| parameter | description |

| AttributeId | The `eAttribute` int (matches the id used by [`GetAttributeName`](./Diablo4Storage/GetAttributeName.md) ‚Äî names are resolved via that overload ‚Äî the compound-key overload that handles the tag-conditional cases the raw id cannot disambiguate; the AttributeId is itself a power-budget category, not a stat key ‚Äî per the FR-C21 finding recorded in `casc-diablo4-format.md ¬ß7.6`). |
| ParamPlus12 | The associated GBID (`0xFFFFFFFF` when the attribute is tag-agnostic). On tag-conditional attribute ids ‚Äî e.g. `AttributeId 259` (`DamageBonusTag`) ‚Äî this GBID identifies the skill-tag the affix scales against (Abyss / Archfiend / Demonology / etc.); the consumer can call [`GetAttributeName`](./Diablo4Storage/GetAttributeName.md) to resolve the per-tag display string (the compound-key lookup in [`LabelByCompoundKey`](./AttributeNames/LabelByCompoundKey.md)) and [`HasParam`](./GlyphAffixAttributeRef/HasParam.md) to filter out the unset slots. |
