# ItemType class

A decoded Diablo IV item base type — an entry in the item-type dictionary (SNO group ItemType = 98, the game's `eItemType` enum: `Sword`, `Amulet`, `Helm`, `Charm`, `HealthPotion`, …). Every [`ItemDefinition`](./ItemDefinition.md) references its base type ([`ItemTypeSnoId`](./ItemDefinition/ItemTypeSnoId.md)); this record is what classifies an item as a weapon / armor / jewelry / charm (LIB-1).

## Remarks

Byte layout (clean-room, `docs/casc-diablo4-format.md §13`): payload base `0x10`; `snoId` at payload `0`. The classification fields (all payload-relative): a kind word at `+0x08` (`32`/`48` ⇒ equippable gear; smaller values ⇒ non-gear), a sub-kind at `+0x0C` (`5` ⇒ Charm), a weapon-family enum at `+0x30` (`≥ 0` ⇒ a weapon-slot item, `-1` ⇒ not a weapon), an armor-value scalar at `+0x3C` (`0` ⇒ jewelry, `> 0` ⇒ body armor), and a slot word at `+0x44` (`> 0` for armor/jewelry). Verified across the full g98 set on build 3.1.1.72836. The type [`Name`](./ItemType/Name.md) is the CoreTOC name — a durable, opaque stable id (Appendix C).
