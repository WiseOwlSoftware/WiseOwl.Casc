# AttributeNames.LabelByCompoundKey property

FR-C28 (CL-85) — compound-key map resolving the tag-conditional attribute names where the same [`LabelByAttributeId`](./LabelByAttributeId.md) entry can't disambiguate (e.g. `AttributeId 259` = `DamageBonusTag` — the same id covers Abyss / Demonology / Conjuration / Hellfire / ... damage, the per-tag identity carried in `ParamPlus12`). Keyed on the raw `(int AttributeId, uint ParamPlus12)` tuple; the value is the resolved display name as the engine renders it in the in-game tooltip (e.g. `(259, 0x32ABA6FB) → "Demonology Damage"`). Surfaced to the consumer via [`GetAttributeName`](../Diablo4Storage/GetAttributeName.md); ParagonNodeInfoBuilder consults it on every node stat row.

## Remarks

Provenance. The map is built from the empirical scan of every `ParagonGlyphAffix` (group 112) and `ParagonNode` (group 106) record on the live build `3.0.2.71886` — every observed `(AttributeId, ParamPlus12)` tuple where `ParamPlus12 ≠ 0xFFFFFFFF` (the "no parameter" sentinel). Tag GBIDs were cracked clean-room via the `Skill_<TagName>` DJB2-lowercase pattern (see `docs/d4-hash-dictionary.md` — 19 confirmed); uncracked tags use the empirical name derived from the per-affix / per-node sno-name convention (`<TagName>Damage_<Class>_<Slot>` on affixes; `Generic_Magic_Damage<TagName>` on nodes) — surfaced as the consumer-visible label even when the engine internal name is not yet hash-recovered.

Base templates. The per-AttributeId base label is determined by the engine's `AttributeDescriptions` entry (sno 4080). E.g. `(259, Tag)` uses `Damage_Percent_Bonus_Per_Skill_Tag = +[...] {VALUE1} Damage` → `"{Tag} Damage"`. `(290, Tag)` uses `Crit_Damage_Percent_Per_Skill_Tag` → `"{Tag} Critical Strike Damage"`. The map bakes the substituted enUS string directly; localisation extension is a future iteration.
