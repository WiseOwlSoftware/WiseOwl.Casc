## Remarks

Byte layout (clean-room, `docs/casc-diablo4-format.md ¬ß11.1` / ¬ß12, Appendix A CL-21): payload base `0x10`; `snoId` at payload `0`; `eClass` (the game's internal class enum ordinal) at payload `16`. The `eClass` ordinal is sparse but stable (Sorcerer 0, Barbarian 1, Rogue 3, Druid 5, Necromancer 6, Spiritborn 7, Paladin 9, Warlock 10) and is the ordering behind the glyph `fUsableByClass` rank (¬ß7.3 / FR-D3).

The stat-conversion map is three `DT_VARIABLEARRAY` descriptors at payload `+0x40`/`+0x50`/`+0x60`, each a single `(coreIndex:int32, weight:float32, ‚Ä¶)` element. In slot order they name the core that feeds Skill Damage (the primary, weight 1.25), Critical Strike Chance, and Resource Generation (weight 1.0). The mapping is per-class and read straight from the record ‚ÄĒ verified against live core-stat tooltips for all four primary-attribute archetypes (Warlock/Rogue/Necromancer/Barbarian). The per-point coefficients are universal engine constants in [`CharacterStatModel`](./CharacterStatModel.md).
