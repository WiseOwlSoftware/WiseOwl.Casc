# AffixDefinition.MaxRank property

FR-C38 (#55, CL-107) — the per-aspect maximum legendary rank: the cap `CurrentLegendaryRank()` reaches in a rank-scaled aspect affix's value formula ([`InlineFormula`](../AffixEffect/InlineFormula.md)), read from the int32 at payload `+0x94`. It is per aspect, not a global constant — the live build (`3.1.1.72836`, Season 14) has caps of 21 (most aspects — 394 of 661), 11 (82), 16 (79), 6 (19), and a spread of others. So an aspect's rolled value span is `[InlineFormula(1) … InlineFormula(MaxRank)]` — e.g. Edgemaster's (`MaxRank` 21, `40+(rank-1)*1`) spans `[40 … 60]`%, and Aspect of Coagulation (`MaxRank` 6, `15+(rank-1)*1`) spans `[15 … 20]`%. Owner in-game oracle (Codex of Power) validated 4/4 aspects exactly (21/21/6/16); the field is present + in a sane rank range on 661/661 `legendary_*` aspect affixes.  The rank floor is 1 (`InlineFormula(1)` = the range minimum). Meaningful for rank-scaled aspect affixes (those whose [`InlineFormula`](../AffixEffect/InlineFormula.md) references `CurrentLegendaryRank()`); for other affixes the field may hold an unrelated quantity or 0, so gate its use on the affix being a rank-scaled aspect. This is season-drift-prone (a build may re-tune caps) — see `casc-diablo4-format.md` §8.2. 0 on a short/malformed record.
