# AffixDefinition.AllowedItemTypes property

FR-C36… (#51, CL-106) — the item types this affix is allowed to roll on: the affix pool from the item's side. A gear affix (magic / rare / legendary) is rolled from a per-item-type pool; this is the affix's list of eligible `eItemType` ids (decoded from the `DT_VARIABLEARRAY[int]` at payload `+0x78`). Semantically verified: `CoreStat_Strength` → armor/jewelry types, `CoreStat_Strength_Weapon` → the weapon types, `Charm_Armor_Percent` → `[71]`. Invert across all affixes for the "what can roll on this type" query (!:Diablo4Storage.RollableAffixes(int, string)).

## Remarks

The values are engine `eItemType` ordinals (the enum, not group-98 `ItemType` SNOs). Names for these ordinals are not resolvable from the SNO data alone (an oracle / EXE enum is needed) — the raw ids are exposed as the byte-verified primitive; resolving them to item-type names is a follow-up. Empty (never `null`) when the affix authors no allowed-type list (e.g. unique/inline-formula affixes fixed to one item).
