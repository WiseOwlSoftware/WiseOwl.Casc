# TiledStyleBinding constructor

FR-C14 R9 — one binding of a UI widget's `snoTiledStyle` field to a [`TiledStyleDefinition`](../TiledStyleDefinition.md). Surfaced on [`TiledStyleBindings`](../ParagonBoardChrome/TiledStyleBindings.md) so consumers can reconstruct the engine's overlay composition for the paragon board.

```csharp
public TiledStyleBinding(string WidgetName, uint WidgetClassId, int TiledStyleSnoId, 
    TiledStyleDefinition? Style)
```

| parameter | description |
| --- | --- |
| WidgetName | The bound widget's name (e.g. `"Vignette"`, `"Board_Info"`, `"Paragon_Points_Container"`). Names are useful for the consumer to attribute the overlay to a specific visual element; semantic-role naming is the consumer's (per memory `feedback_widget-name-not-role`). |
| WidgetClassId | The bound widget's class id (28-bit truncated DJB2 of the class name). `0x1E3077C7` is the standard "draw a textured rect" widget class (Background, Framing, Vignette, Divider …); `0x112661D5` is the Stack/Layout container class (Layout_Stack, ControllerStack, Board_Selector_BG …). |
| TiledStyleSnoId | The SNO id the widget's `snoTiledStyle` field is bound to. Pass to [`ReadTiledStyle`](../Diablo4Storage/ReadTiledStyle.md) to read the full record; [`Style`](./Style.md) is the pre-read convenience. |
| Style | The pre-read [`TiledStyleDefinition`](../TiledStyleDefinition.md), or `null` if the SNO id couldn't be read (typically because it's a sentinel value like `1`, `3`, `20` — observed as "enum-shaped" small-id bindings whose semantics are not the same as a real SNO reference). |
