# TiledStyleDefinition.WindowPieces property

FR-C14 R10 / FR-C19 — for the `TiledWindowPieces` variant, the 9 authored piece handles at blob `+0x60..+0x80`, in the record's array order (index 4 resolves to the centre fill; the other 8 to the border pieces — observed as quadrant/edge brackets, not pre-cut 3×3 thirds). Each is a texture frame handle (resolve via the icon-frame index). The record carries the piece handles + [`ImageScale`](./ImageScale.md) + the tile flags only — NOT the per-piece crop/placement geometry. The exact composition (how the engine crops/anchors/blends these 9 pieces into the border) lives in the engine UI code and is not recoverable from this record; a naïve corner/edge/centre 3×3 placement overlaps the brackets. Treat this as the authored asset set; the composition must be calibrated against the live render (see FR-C19 #30). Empty for non-window-pieces variants (use [`SourceImageHandle`](./SourceImageHandle.md) + the NSlice fields).
