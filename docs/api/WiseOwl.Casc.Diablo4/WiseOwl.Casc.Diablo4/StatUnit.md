# StatUnit enumeration

How a [`FlatValue`](./ParagonNodeStat/FlatValue.md) should be rendered for display. Flat means "show as a raw number" (`+5 Strength`); Percent means "show with a percent sign" (`+7.5% Total Armor`); Multiplier is reserved for future stats that display as a multiplicative factor (`×1.5`) — no paragon node uses it today.

## Values

| name | value | description |
| --- | --- | --- |
| Flat | `0` | Render as a raw number (`+5 Strength`, `+3 All Resistance`). |
| Percent | `1` | Render as a percentage (`+7.5% Total Armor`, `+10% Damage`, `+4% Max Life`). |
| Multiplier | `2` | Render as a multiplicative factor (`×1.5`). Reserved; not used by any paragon node observed. |

## Remarks

The library's heuristic derives the unit from the stat token + the underlying [`AttributeId`](./NodeAttribute/AttributeId.md) (best-effort, matches the in-game tooltip on every node we've validated). It is a hint — the [`FlatValue`](./ParagonNodeStat/FlatValue.md) is the numeric truth.
