# ItemDefinition.SnoName property

The item's CoreTOC name — the durable, opaque stable id (e.g. `"Amulet_Unique_Rogue_100"`), distinct from the localized [`Name`](./Name.md). Populated by [`ReadItem`](../Diablo4Storage/ReadItem.md) and [`EnumerateItems`](../Diablo4Storage/EnumerateItems.md); Empty on the byte-only [`Parse`](./Parse.md) (no [`CoreToc`](../CoreToc.md)). De-duping uniques (#56). The live Item group carries leftover seasonal/PTR duplicate unique SNOs whose CoreTOC name is season-prefixed — `^S\d+_` (e.g. `S10_Amulet_Unique_Rogue_100_Boots`, a stale duplicate of `Amulet_Unique_Rogue_100` with a different slot / more explicits). They share the localized display [`Name`](./Name.md) with the canonical item, so any surface that de-dupes uniques by display name must prefer the non-season-prefixed `SnoName` (the `^S\d+_` test is the reliable signal — see `casc-diablo4-format.md §11.4`). No clean structural bit separates them (the `0x10000000` flag correlates ~91% but with exceptions).
