# SelectionHighlight constructor

FR-C19 — the engine's mouse-over / cursor selection highlight, surfaced as the set of authored recipes that draw it. The selection highlight is orthogonal to, and not part of, the paragon node render recipe ([`ParagonNodeRecipe`](../ParagonNodeRecipe.md)): a node can be selected whether or not it is purchased, so the consumer composes the node from its recipe and then, if the node is selected, draws the matching selection style on top.  Crucially, the game does not require the consumer to assemble the highlight from loose atlas frames — it authors each selection shape as a [`TiledStyle`](../TiledStyleDefinition.md) 9-slice recipe (group UiStyle = 103) over the `2DUI_SelectionHighlight` / `2DUITiled_SelectionHighlight` atlases. This accessor returns those authored recipes ([`Styles`](./Styles.md)) so the consumer applies the existing TiledStyle machinery ([`ReadTiledStyle`](../Diablo4Storage/ReadTiledStyle.md)) — exactly the path used for every other tiled UI drawing — and performs no composition or shape-classification of its own.  Each style carries its authored name and the [`SelectionShape`](../SelectionShape.md) derived from it (e.g. `ControllerSelectionCircle` → Circle, `SelectionRectangleInset` → Rectangle): the consumer selects the style whose shape matches the node's silhouette. The shape comes from the engine's own recipe name, not from an atlas-frame-size guess.
