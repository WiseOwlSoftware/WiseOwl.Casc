# ParagonPowerBudget class

The six budget-multiplier intrinsics that paragon-node magnitude formulas multiply against. Empirically pinned (clean-room) from owner-validated in-game readings on build `3.0.2.71886`; the engine implements these as formula-DSL intrinsic functions (`ParagonPowerBudgetMultiplierNode<Rarity>{Major|Minor}<Off|Def>()`) that are absent from every shipped GameBalance data table — not in [`AttributeFormulaTable`](./AttributeFormulaTable.md) (sno `201912`), not in any `PowerFormulaTables`, and no SNO carries "Budget" in its name. So the library bakes them as a calibration table.

## Remarks

The magnitude model (FR-C21, `docs/casc-diablo4-format.md` §7.2 + Appendix C): a paragon stat node's displayed magnitude is `formula-constant × budget-multiplier`. Worked verifications against in-game readings:

* `Generic_Magic_Armor`: `0.75 × MagicDefensive (10) = 7.5%` ✓
* `Generic_Magic_DamageToElite`: `3 × MagicOffensive (2.5) = 7.5%` ✓
* `Generic_Rare_AllResistance`: `0.75 × RareMajorDefensive (4) = 3.0%` ✓
* `Generic_Rare_MaxLife`: `1 × RareMajorDefensive (4) = 4.0%` ✓
* `Generic_Rare_Damage`: `2 × RareMajorOffensive (5) = 10%` ✓
* `Warlock_Rare_006` (Demonology tag): `3.5 × RareMajorOffensive (5) = 17.5%` ✓
* `Generic_Rare_CriticalDamage`: `3 × RareMinorOffensive (5) = 15%` ✓

Re-verification. If a future Diablo IV build appears to disagree with displayed magnitudes, the cause is one of: (a) the per-node formula-constant changed (re-decode the [`NodeAttribute`](./NodeAttribute.md) / [`AttributeFormulaTable`](./AttributeFormulaTable.md) entry), or (b) the engine retuned a multiplier. The library has no way to read the multipliers from data — pin them empirically and document the build they were last validated against here.

Normal-rarity nodes (Common) use plain numeric constants — their formulas do not invoke a budget multiplier intrinsic at all (e.g. `"5"` for `ParagonNodeCoreStat_Normal`). The evaluator returns the constant directly in that case.
