# Diablo4Storage.TryGetDataAttributeName method

FR-C32 (CL-93) — resolve a flag-namespaced`AttributeId` (a negative id, high bit `0x80000000` set) to its `DataAttributes` designer-table name. These refs — carried on paragon nodes, glyph affixes, and item affixes for conditional/seasonal/per-power bonuses (Berserking, Shadowform, Demonform, Volatile, kill-streak, …) — are not engine `eAttribute` ids and do not resolve through [`GetAttributeName`](./GetAttributeName.md) (a disjoint namespace); this method reads them from the data-defined `DataAttributes` table (SNO `1907204`) by ordinal `attributeId & 0x7FFFFFFF`.

```csharp
public bool TryGetDataAttributeName(int attributeId, out string name)
```

| parameter | description |
| --- | --- |
| attributeId | The raw (possibly flag-namespaced) `eAttribute` int, exactly as carried on [`AttributeId`](../NodeAttribute/AttributeId.md) / [`AttributeId`](../GlyphAffixAttributeRef/AttributeId.md) / [`AttributeId`](../AffixEffect/AttributeId.md). |
| name | The resolved `DataAttributes` token, or Empty. |

## Return Value

`true` iff *attributeId* is a flagged ref that resolved to a table entry.

## Remarks

The returned *name* is the table's authored token (e.g. `"Warlock_Demonform_Damage_Bonus"`, `"Multiplicative_Damage_Percent_Bonus_While_Volatile"`, `"Barb_Berserking_AttackSpeed"`) — the first-party designer name, not a localized display string (these conditional attributes have no standalone tooltip label; the consumer owns any humanization). The additive/multiplicative split is in the token (a `Multiplicative_` prefix distinguishes the pair member).

Returns `false` — with *name* set to Empty — for a non-flagged (positive) engine id, the `-1` "no attribute" sentinel, an ordinal outside the table, or when the `DataAttributes` record is unreadable. Verified: item-affix ordinal `84 = Barb_Berserking_AttackSpeed`; node/glyph ordinal `251 = Warlock_Demonform_Damage_Bonus`, `252 = Multiplicative_Warlock_Demonform_Damage_Bonus` — see `casc-diablo4-format.md §11.3`.
