# Diablo4Storage.ReadItemTypeNames method

The engine `eItemType`ordinal → base-type name map (#51, CL-108) — the key that names an affix pool. An affix's [`AllowedItemTypes`](../AffixDefinition/AllowedItemTypes.md) (and the inverse [`RollableAffixes`](./RollableAffixes.md)) speak these ordinals; this resolves each to a readable base-type name (`16 → "Helm"`, `71 → "Charm"`, `1 → "Axe"`). Built from the g98 [`ItemType`](../ItemType.md) records ([`EItemType`](../ItemType/EItemType.md)). Where several base types share one ordinal (1H/2H/class variants — `Axe` and `Axe2H` are both `1`) the representative is the shortest equippable name (the base). Cached after the first call.

## Remarks

A few ordinals seen in affix pools (e.g. `9`, `23`) have no g98 record and are therefore absent from the map — they are engine-aggregate/legacy values not nameable from the data.
