# Diablo4Storage.TryReadParagonNodeTitle method

Resolve a `ParagonNode`'s engine user-facing tooltip-title via the §6.7 sibling-StringList convention: `ParagonNode_<NodeSnoName>` in group StringList, label `Name`. The engine authors a sibling for every node that has its own tooltip header — structural nodes (`StartNodeBarb` / `StartNodeWarl`/etc. → `"Paragon Starting Node"`, `Generic_Gate` → `"Board Attachment Gate"`), class-specific rare nodes (`Warlock_Rare_006` → `"Binding"`, etc.), and named legendary nodes. Only the generic `Generic_<Rarity>_<Token>` stat-node family (the stat-shop of magic / rare-minor / rare-major / legendary affixes shared across classes — `Generic_Magic_DamageToElite`, `Generic_Magic_Armor`, etc.) has no sibling — the tooltip on those is composed by the engine from the stat-token + the rarity, not from an authored display name. Returns `false` + Empty for the no-sibling case (honest sentinel; consumer composes from [`Stats`](../ParagonNodeInfo/Stats.md) / [`Kind`](../ParagonNodeInfo/Kind.md)).
