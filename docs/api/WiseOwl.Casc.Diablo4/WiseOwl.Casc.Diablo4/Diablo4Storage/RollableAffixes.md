# Diablo4Storage.RollableAffixes method

#51 (CL-106) — the inverted affix pool query: every gear affix that can roll on the given `eItemType` ordinal (its [`AllowedItemTypes`](../AffixDefinition/AllowedItemTypes.md) contains *itemTypeId*). The convenience built on the per-affix [`AllowedItemTypes`](../AffixDefinition/AllowedItemTypes.md) primitive — "what can roll on this item type."

## Remarks

Lazy; each yielded affix is decoded byte-only (identity + [`Effects`](../AffixDefinition/Effects.md) + [`AllowedItemTypes`](../AffixDefinition/AllowedItemTypes.md), no localized `Name`/`Desc` — call [`ReadAffix`](./ReadAffix.md) with the affix's [`SnoId`](../AffixDefinition/SnoId.md) for those). A full pass over the group-104 affix group; cache the result if you query many types (the pool is stable per build).
