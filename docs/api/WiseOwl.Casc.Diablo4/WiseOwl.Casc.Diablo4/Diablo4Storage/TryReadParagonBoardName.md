# Diablo4Storage.TryReadParagonBoardName method

Resolve a `ParagonBoard`'s localized display name — the in-game board name ("Start", "Dynamism", "Pyrosis", …) — for a locale.

```csharp
public bool TryReadParagonBoardName(int boardSnoId, out string name, string locale = "enUS")
```

| parameter | description |
| --- | --- |
| boardSnoId | The `ParagonBoard` SNO id (group 108). |
| name | The localized board name, or Empty. |
| locale | Locale (default [`DefaultLocale`](./DefaultLocale.md)). |

## Return Value

`true` iff a localized name was decoded.

## Remarks

The board's localized name is not on [`ParagonBoardDefinition`](../ParagonBoardDefinition.md) (group 108) at all; it lives in the board's sibling StringList table (group StringList = 42). The D4 convention, recovered clean-room and recorded in `docs/casc-diablo4-format.md §6.4` (Appendix A CL-15): the sibling table's CoreTOC name is `"ParagonBoard_" + boardSnoName` (e.g. board `Paragon_Warlock_00` → table `ParagonBoard_Paragon_Warlock_00`), and the localized string is under label `"Name"`. The SNO ids are unrelated (no fixed offset — Warlock happens to be board−1, Sorcerer is not); resolution is strictly name-keyed via [`CoreToc`](./CoreToc.md). Holds for every class (Barb/Druid/Necro/Paladin/Rogue/Sorc/Spirit/Warlock).

Raw decoded value only — no fallback policy. If the board SNO name is unknown, the sibling table is absent, or it carries no `Name` label, this returns `false` and the consumer owns the fallback (e.g. show the SnoName identifier).
