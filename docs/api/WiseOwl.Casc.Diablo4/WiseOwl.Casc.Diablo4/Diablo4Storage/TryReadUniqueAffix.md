## Remarks

A unique item (group Item = 73, e.g. `1HAxe_Unique_Druid_100`) shares its SNO name verbatim with an affix definition (group Affix = 104) of the same name вЂ” the generalized В§6.7 sibling convention (as with the localized sibling-StringList tables, CL-20). The item record itself references only its model actor and base-item template, not the affix, so the affix is reached by that shared name. This wires the two already-decoded readers ([`ReadItem`](./ReadItem.md) + [`ReadAffix`](./ReadAffix.md)): pass a unique item's SNO id and get its power's decoded affix. Name-verified across the unique roster; returns `false` when the item has no same-name affix (a non-unique item, or a seasonal `SNN_`-prefixed variant whose affix name differs).
