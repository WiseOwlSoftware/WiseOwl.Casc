# ParagonMagnitudeFormula.TryEvaluate method

FR-C33 (CL-98) — evaluate *formulaText* and distinguish an unsupported engine function from a value that legitimately computes to NaN (the gap [`Evaluate`](./Evaluate.md) can't express — it returns a silent NaN for both).

```csharp
public static bool TryEvaluate(string formulaText, out double value)
```

| parameter | description |
| --- | --- |
| formulaText | The formula source text. |
| value | The computed value, or NaN when an unsupported function short-circuited the evaluation. |

## Return Value

`true` iff the expression contains no unsupported function call (its *value* is a genuine computed result).

## Remarks

The only calls this evaluator implements are the six zero-arg budget-multiplier intrinsics (Double})). Any other engine function — the affix roll functions (`FloatRandomRangeWithInterval`, `RandomInt`, `IPower`, `ROUND`, …) or a player-state accessor — is unsupported: `TryEvaluate` returns `false` so the consumer branches deliberately (e.g. renders the raw formula text) rather than propagating a silent NaN into a displayed number. A pure arithmetic / budget-multiplier expression returns `true` with its value (including a genuine NaN from e.g. `0/0`, which is a computed result, not an unsupported call). Throws on syntactically invalid text, as [`Evaluate`](./Evaluate.md) does.
