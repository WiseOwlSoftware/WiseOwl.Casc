# Devlog 0046 — FR-C16 R9 / FR-C18: node-template child sub-records decode structurally

*2026-05-21*

## Context

After CL-44/46/47/48 shipped the paragon node-render recipe (released in
`v0.3.0-alpha`), the Optimizer's live render against the full board
surfaced four recipe-structure issues (#26), and a new request (#29 /
FR-C18) reported the rarity `Template_Node_{Magic,Rare,Legendary}`
`WidgetRect` decoding all-zero (the colored disc oversized vs the grey
`Node_IconBase`). Both were `awaiting:casc`.

## One root cause

All five symptoms trace to the same loss: CL-46/47/48 harvested a parent
template's anonymous child sub-records only as a **flat handle soup**
(`UiWidget.ExtraLayerValues` → `ParagonNodeRecipeLayer.CompositeHandles :
uint[]`), discarding each child's authored rect.

The decode (recon: `build/SnoScan -- noderecon 657304`) shows each
anonymous child is a **self-contained name-less mini-widget**: a class id +
`0xFFFFFFFF` sentinel, then its own schema run + positionally-keyed value
records (0x22 or tag-2), bounded by the next sibling marker. So the
handle↔rect pairing is right there in the bytes — the flat harvest threw
it away.

## Findings (scene 657304)

- **FR-C18 #29** — the `Template_Node_<rarity>` *parent* binds no usable
  rect (Magic: `bActive,nBottom=0,nRight=0`; Rare: nothing; Legendary:
  `nBottom=0,nRight=0`). The all-zero parent rect is **faithful**. The disc
  geometry is on the children: Magic/Rare selected disc = inset **7** (=
  `Node_IconBase`, 86²), Legendary = inset **−3** overscan. The
  unselected/selected pair is co-sized.
- **#26.1** — the rarity disc child shares `Node_IconBase`'s inset-7
  geometry ⇒ the rarity disc **substitutes the base disc**; it is not a
  z=121 top layer. Blob order is draw order for the shared state-widget run
  (96–120), but the `Template_Node_*` (121–130) are mutually-exclusive
  variant definitions appended after — the chosen template's disc draws at
  the base slot, with the symbol (`Node_Icon`, z=106) on top.
- **#26.2** — `Template_Node_Socketable` is authored empty (240-byte stub:
  0 children/fields/handles). Faithful; socket art = `Node_Glyph` /
  `Usage_Slot_*` (`0x3084D186` bead).
- **#26.3** — `Template_Node_Starter` filigree `0xA0F996FE` = 140² at −18
  overscan (surrounds the base); base `0xF8312CA8` inherits the cell. Bug
  was the consumer drawing both full-cell.
- **#26.4** — `Template_Node_Quest` IS the gate: filigree `0xA0F996FE`
  (−20), ornate `0xC2DF4786`/`0x0E6B6249` (inset 3), conditional locator
  `0x6D68F45F` (inset 22/26/24/24). Owner's diamond
  `0xACDA0144`/`0x61FB8387` is not bound anywhere in 657304.

## Shipped (CL-50)

- `UiScene.Parse` decodes child sub-records into `UiWidget.Children`
  (`UiWidgetChild{ClassId, Fields}`), one level deep. `ExtraLayerValues`
  retained for the §10.14 losslessness gate.
- `ParagonNodeRecipeLayer.CompositeHandles : uint[]` → `CompositeLayers :
  IReadOnlyList<NodeDiscLayer>`; `NodeSelectionDiscs.{Unselected,Selected}`
  go `uint` → `NodeDiscLayer{ImageHandle, Rect, Active}`. The rarity disc
  pair is co-sized (an empty-rect side inherits its sibling's inset).
- §10.3 grammar + Appendix A CL-50 updated. Recon command `noderecon`
  added to `build/SnoScan`.

57/57 tests green on build `3.0.2.71886` (the recipe acceptance test
extended with the disc/filigree/locator rects + the empty socketable).

## Boundary note

CASC surfaces the authored per-child rects + the substitution evidence
(shared inset-7 geometry). The z-order semantics for the
mutually-exclusive `Template_Node_*` variants are documented engine
behavior (the disc substitutes `Node_IconBase`), not a decoded z field —
no fabrication. The consumer keeps its `WidgetName → predicate` map and now
draws each layer at its own rect.
