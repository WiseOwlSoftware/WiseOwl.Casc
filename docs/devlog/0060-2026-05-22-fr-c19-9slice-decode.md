# 0060 — FR-C19 #30: TiledWindowPieces 9-slice fully decoded

2026-05-22 · CL-62 · branch `fr-c19-9slice-decode`

## Trigger

Owner visual-close (relayed via #30): the mouse-over highlight appears but shows
**only the top-left corner/sides**, aligned to the disc edge, not framing the
node square. Exactly the partial-decode caveat I flagged: stretching the single
`SelectionRectangleInset` (585031) source frame full-cell doesn't compose a
border. Ask: finish the TiledStyle decode so a real 9-slice can render.

## Finding

`SelectionRectangleInset` is the **`TiledWindowPieces`** variant (typeTag
`0x02E46583`) — *not* NSlice, so the old parser left it `partial=True` with no
usable composition. Dumping the blob tail (`f32 103 585031`):

```
+0x50 typeTag = 0x02E46583  (TiledWindowPieces)
+0x60 .. +0x80 = 9 consecutive u32 handles
```

Resolving those 9 handles (`framesize`):

```
idx  off    handle      atlas / native            role
 0   +0x60  0x95DA4E78  585030  64²               TL corner
 1   +0x64  0x5192E52B  585030  64²               T edge
 2   +0x68  0xEA71A5AD  585030  64²               TR corner
 3   +0x6C  0xB1C206BA  585030  64²               L edge
 4   +0x70  0xD756FD92  141461 (BackgroundSquares) 100²  CENTRE fill
 5   +0x74  0x7DBD2725  585030  64²               R edge
 6   +0x78  0x706AB7ED  585030  64²               BL corner
 7   +0x7C  0x35D0D378  585030  64²               B edge
 8   +0x80  0xCB0C9F2F  585030  64²               BR corner
```

Index 4 = the centre square (different atlas, 100²); the other 8 = the 64²
border slices. That's a textbook 9-slice stored as explicit piece frames in
**row-major 3×3** order `[TL,T,TR,L,C,R,BL,B,BR]` — no inset arithmetic needed.

## Decoded

`TiledStyleDefinition.WindowPieces` (`IReadOnlyList<uint>`) — the 9 piece handles
for the `TiledWindowPieces` variant (`+0x60..+0x80`), empty otherwise. Decoding
them clears `HasPartialDecode` for the variant. `SourceImageHandle` (`+0x60`)
stays = `WindowPieces[0]` (the TL corner) for back-compat.

Consumer renders the 9-slice: corners at native size × `ImageScale` in the cell
corners, edges stretched between, centre filling the interior — framing the full
node square (not the disc edge).

## Acceptance

`ReadTiledStyle_decodes_TiledWindowPieces_9slice`: typeTag = TiledWindowPieces,
`HasPartialDecode == false`, 9 pieces, `[0] == SourceImageHandle`, `[4] ==
0xD756FD92` (centre), every piece resolves to a frame. 52/52 Diablo4 tests green
on `3.0.2.71886`.

## Note

This generalises to the other `*WindowPieces` selection styles. The NSlice path
(tile flags at `+0x88`) is unchanged. The Optimizer can now switch the #30 hover
cursor from stretch → 9-slice composition.
