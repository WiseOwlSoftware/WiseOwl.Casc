# 0081 — FR-C24 Headhunter counter-round: ParagonGlyph_<SnoName> sibling

2026-05-24 · CL-86 · branch `fr-c24-headhunter`

## Trigger

`casc-fr#36` counter-round from the Optimizer (2026-05-24): one
remaining defect after CL-83/CL-84 consume-verify. Glyph sno `2117207`
`Rare_Will_Generic` returns `LocalizedTitle = ""`; in-game oracle
(`Screenshots/calibration/Glyphs/14_Headhunter.png`) shows
`"Headhunter"`. Structurally a `Rare_<Stat>_Generic` naming pattern
that the §6.7 sibling-StringList convention misses.

## What I went looking for

Which StringList carries the `"Headhunter"` string for this glyph.

Two natural candidates:

```text
Item_ParagonGlyph_Rare_Will_Generic   ← CL-79's convention; predicted source
ParagonGlyph_Rare_Will_Generic        ← non-prefixed variant
```

## What I found

**Only the non-prefixed table exists** for this glyph:

```
group 42  id 2117206  ParagonGlyph_Rare_Will_Generic   (label Name = "Headhunter")
                      (no Item_ParagonGlyph_Rare_Will_Generic sibling)
```

For comparison, a numbered glyph (e.g. `Rare_011_Intelligence_Side` =
"Guzzler") has **both**:

```
group 42  id 1026907  Item_ParagonGlyph_Rare_011_Intelligence_Side  (Name = "Glyph: Guzzler" + Description)
group 42  id 1111836  ParagonGlyph_Rare_011_Intelligence_Side       (Name = "Guzzler")
```

The `Item_`-prefixed table is the consumer-facing item-tooltip variant
(carries the inventory "Glyph: <Title>" form + a "Right-click to
learn" Description). The non-prefixed table is the canonical engine
title source (bare title, no `Glyph:` prefix). The `_Generic` shape
only emits the engine source.

CL-79's convention picked the wrong one. CL-86's fix: pick the
non-prefixed source.

## Fix

```diff
- private const string ParagonGlyphStringTablePrefix = "Item_ParagonGlyph_";
- private const string GlyphTitlePrefix = "Glyph: ";
+ private const string ParagonGlyphStringTablePrefix = "ParagonGlyph_";

  // ...
- title = raw.StartsWith(GlyphTitlePrefix, StringComparison.Ordinal)
-     ? raw[GlyphTitlePrefix.Length..]
-     : raw;
+ title = raw;
```

The non-prefixed table carries the bare title directly so the
universal `"Glyph: "` prefix-strip from CL-79 is no longer needed
— removing it simplifies the code path. All 21 Warlock glyphs (and
every other class's glyphs) decode correctly through the single
non-prefixed convention.

## Boundary call: why swap, not branch

The CL-79 path could have been kept as the primary with a fallback to
the non-prefixed table when the `Item_`-prefixed one is missing.
Picking that path would have left two equivalent sources active
(every numbered glyph has both tables); the non-prefixed table is
strictly more universal and carries the canonical bare title, so the
swap is the cleaner one-source decision. The `Item_`-prefixed table
remains the right source for an in-inventory item-tooltip rendering
(when a future FR asks for the "Glyph: <Title>" item-tooltip form +
the localized `Description` text); CL-86 surfaces only the bare
engine title.
