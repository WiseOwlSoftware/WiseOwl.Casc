## The fix (CL-47)

- **`ParagonNodeRecipeLayer.SelectionDiscs: NodeSelectionDiscs?`** â€” for
  the three rarity sub-templates, the first two resolvable composite
  handles split into `Unselected`/`Selected`. The interpreter draws
  `Selected` only on the currently-selected node. `null` for non-rarity
  layers. The interior-fill extras stay in `CompositeHandles`.
- **Resolvable-only composites.** `NodeRecipe` now takes the
  `IsParagonTextureHandle` icon-catalog validator; a composite handle
  must resolve to an atlas frame. The negative insets (and any other
  non-frame word) are excluded.
- **Implausible-rect guard.** A projection-level guard zeroes rect field
  values beyond Â±4096 (the design canvas is 1920Ă—1200), so a mis-keyed
  handle never propagates as an inset. `Node_Icon` is the runtime-filled
  symbol slot (`HIconMask`, fit-to-disc-centre) â€” its template rect is
  not load-bearing.

The proper fix for defect #3 is a re-RE of the sparse instance-binding

This guard contains the symptom now without that risk.
